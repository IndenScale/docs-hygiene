# 身份与依赖

DH 把项目文档看作带有稳定身份和显式关系的内容，而不是一组只能通过路径找到的文件。

## 路径不是身份

路径回答“文件现在放在哪里”，身份回答“这个对象是谁”。需求从 `drafts/` 移到 `product/`，或者因为本地化增加另一个文件时，语义对象可能仍然是同一个。

稳定身份使这些操作成为可检查的迁移，而不是一次悄无声息的断链。

## 导航链接与语义引用

标准 Markdown Link 用于导航：

```markdown
[配置说明](../guide/configuration.md)
```

语义引用声明“当前资产消费了另一个稳定身份的含义”：

```markdown
[[DH-GOVERNANCE-EDGE|治理边]]
```

前者只需要目标可达，后者还需要身份、期待类型、生命周期和治理关系兼容。DH 不会因为一个链接存在就擅自推断业务依赖。

## Reference 与 Pin

Reference 表示依赖关系，Pin 表示“我审阅的是上游的这个确定状态”。

```text
消费资产 ── Reference ──▶ 权威资产
消费资产 ── Pin ────────▶ 已审阅的上游状态
```

上游变化后：

- Reference 用于计算潜在影响；
- Pin 用于确定消费者是否失效；
- 项目策略决定是否必须复核。

不是每条引用都需要 Pin。只有要求精确新鲜度证据的关键依赖才应锁定，否则会制造不必要的维护成本。

## 生命周期

身份不会因为内容停止生效就消失。`superseded`、`archived` 或 `abandoned` 等状态保留历史，同时阻止终止身份继续充当当前权威。替代关系应显式指向新的稳定身份。

## 本地化表示

canonical 与 localized 是同一语义资产的不同语言表示。它们共享身份、生命周期和治理关系；本地化不会产生第二份权威。
