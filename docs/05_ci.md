# CI

Docs Hygiene is designed to run as a local command and as a CI gate.

## Local Check

The default command exits non-zero for errors and leaves warnings advisory. This is the appropriate mode for `seed` and `growing` repositories that have not promoted every document contract to a blocking gate.

```bash
cargo run -- check
```

Maintained repositories can promote warnings to failures:

```bash
cargo run -- check --fail-on-warning
```

For installed binaries:

```bash
docs-hygiene check --fail-on-warning
```

## JSON Output

Use JSON when another tool needs to consume diagnostics. The JSON report uses
a versioned, LSP-style diagnostic shape while remaining a CLI output contract.

```bash
docs-hygiene check --format json
```

Top-level fields:

- `schemaVersion`
- `summary`
- `diagnostics`

Each diagnostic includes `source`, `code`, `severity`, `message`, `path`,
`uri`, `range`, and `relatedInformation`.

## Test Suite

The Rust test suite covers the policy engine directly:

```bash
cargo test
```

Current tests cover required files, bad names, duplicate and missing numbers,
length budgets, language thresholds, document contracts, maturity advice,
concept foreign keys, adapter failures, and suppression matching.

CLI integration tests cover `--help`, `--version`, `scaffold`, and language
policy CRUD commands.

## GitHub Actions

A minimal GitHub Actions step can run the binary after Rust setup.

```yaml
- name: Check docs hygiene
  run: cargo run -- check --fail-on-warning
```
