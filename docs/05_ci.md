# CI

Docs Hygiene is designed to run as a local command and as a CI gate.

## Local Check

The default command exits non-zero for errors and leaves warnings advisory. This is the appropriate mode for `seed` and `growing` repositories that have not promoted every document contract to a blocking gate.

```bash
cargo run -- check
```

Maintained repositories can promote warnings to failures:

```bash
cargo run -- check --fail-on-warning
```

For installed binaries:

```bash
docs-hygiene check --fail-on-warning
```

Profile targets are an independent optional gate:

```bash
docs-hygiene profile --fail-below-target
```

This gate evaluates invariant conformance, not diagnostic severity. A disabled,
suppressed, partial, or missing invariant cannot satisfy a maturity target.

## JSON Output

Use JSON when another tool needs to consume diagnostics. The JSON report uses
a versioned, LSP-style diagnostic shape while remaining a CLI output contract.

```bash
docs-hygiene check --format json
```

Its top-level fields are `schemaVersion`, `summary`, and `diagnostics`. Each
diagnostic includes `source`, `code`, `severity`, `message`, `path`, `uri`,
`range`, and `relatedInformation`.

Rule activation has a separate versioned JSON contract:

```bash
docs-hygiene explain-rules --format json
```

It reports `docs-hygiene.rule-activation.v1`, the project fact snapshot, and
ordered rule decisions without executing external Adapters.

The combined profile contract includes the same facts and decisions plus
document-template binding coverage, the normalized governance graph, Fan and
cycle metrics, dimensional maturity, and invariant evidence:

```bash
docs-hygiene profile --format json
```

Its schema is `docs-hygiene.profile.v1`; top-level fields are `schemaVersion`,
`facts`, `decisions`, `documentTemplates`, `governanceGraph`, `dimensions`,
`overallObserved`, and `meetsTargets`.

Template lifecycle can be checked without writing policy:

```bash
docs-hygiene migrate-templates --check --format json
```

The migration schema is `docs-hygiene.template-migration.v1`. Without
`--check`, compatible and missing revision pins are updated atomically; any
blocked binding prevents the policy file from being written.

Kind Schema and Template pins can be checked and migrated as one atomic plan:

```bash
docs-hygiene migrate-kinds --check --format json
```

The report uses `docs-hygiene.kind-migration.v1`. Any incompatible document or
Template binding prevents both document and policy writes.

The Rust test suite covers the policy engine directly:

```bash
cargo test
```

this repository's real `docs-hygiene.yml` and requires its native policy and
governance graph to pass.

## GitHub Actions

`.github/workflows/ci.yml` runs the complete test suite and Clippy. The test
suite includes the repository dogfood gate, so a broken horizontal reference or
vertical derivation blocks the workflow.

```yaml
- name: Test, including repository dogfood policy
  run: cargo +1.85.0 test --locked
```

The dogfood test disables external adapters because their executables are
environment dependencies; local `cargo run -- check --fail-on-warning` continues
to exercise both native policy and configured adapters.
