# CI

Docs Hygiene is designed to run as a local command and as a CI gate.

## Local Check

The default command exits non-zero for errors and leaves warnings advisory. This is the appropriate mode for `seed` and `growing` repositories that have not promoted every document contract to a blocking gate.

```bash
cargo run -- check
```

Maintained repositories can promote warnings to failures:

```bash
cargo run -- check --fail-on-warning
```

For installed binaries:

```bash
docs-hygiene check --fail-on-warning
```

## JSON Output

Use JSON when another tool needs to consume diagnostics. The JSON report uses
a versioned, LSP-style diagnostic shape while remaining a CLI output contract.

```bash
docs-hygiene check --format json
```

Top-level fields:

- `schemaVersion`
- `summary`
- `diagnostics`

Each diagnostic includes `source`, `code`, `severity`, `message`, `path`,
`uri`, `range`, and `relatedInformation`.

## Test Suite

The Rust test suite covers the policy engine directly:

```bash
cargo test
```

Current tests cover required files, bad names, duplicate and missing numbers,
length budgets, language thresholds, document contracts, maturity advice,
concept foreign keys, governance manifests, horizontal references, vertical
derivations, Library projections, adapter failures, and suppression matching.

CLI integration tests cover `--help`, `--version`, `scaffold`, language policy
CRUD commands, and blocking governance-graph failures. `tests/dogfood.rs` loads
this repository's real `docs-hygiene.yml` and requires its native policy and
governance graph to pass.

## GitHub Actions

`.github/workflows/ci.yml` runs the complete test suite and Clippy. The test
suite includes the repository dogfood gate, so a broken horizontal reference or
vertical derivation blocks the workflow.

```yaml
- name: Test, including repository dogfood policy
  run: cargo +1.85.0 test --locked
```

The dogfood test disables external adapters because their executables are
environment dependencies; local `cargo run -- check --fail-on-warning` continues
to exercise both native policy and configured adapters.
