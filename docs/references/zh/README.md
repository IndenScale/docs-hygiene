# 产品参考层

本目录是 Docs Hygiene 产品意图的稳定参考层。需求应消费这里的定义，不在局部
重新定义同一概念。

## 权威边界

- [通用语言](01_ubiquitous_language.md)定义文档、PRD、测试和未来 Adapter 共同
  使用的产品术语、动作、不变量、结果和用户收益。
- 已实现行为仍以 CLI、配置模型、测试和 `docs/` 下的规则说明为准。
- UL 中标记为规划中的术语只表达产品意图，不代表能力已经交付。

## 变更规则

新产品概念必须先进入 UL，或在 PRD 中声明为局部概念或变更提案。需求可以细化
共享概念，但不能静默赋予它相互竞争的含义。
