# Problem

The current contract maturity, progressive rule activation, graph checks, and
tooling features describe related governance behavior through separate models.
A single six-step ladder would also make precision and scale appear universally
sequential even when a small project needs fine-grained dependencies without
topology budgets. Projects need a compact maturity vocabulary without losing
independent capability results or explicit enforcement authority.
