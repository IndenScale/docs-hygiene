# CI

Docs Hygiene 设计为既能本地运行，也能作为 CI 门禁运行。

## 本地检查

默认命令遇到 error 时返回非零状态，warning 保持为建议。这适合尚未把所有文档契约提升为阻断门禁的 `seed` 和 `growing` 项目。

```bash
cargo run -- check
```

进入维护阶段的项目可以让 warning 同样失败：

```bash
cargo run -- check --fail-on-warning
```

如果已经安装二进制：

```bash
docs-hygiene check --fail-on-warning
```

画像目标可以作为相互独立的可选门禁：

```bash
docs-hygiene profile --fail-below-target
```

这项门禁评估不变量符合性，而不是诊断严重程度。disabled、被 suppression 隐藏、部分
实现或缺失的不变量都不能满足成熟度目标。

## JSON 输出

当其他工具需要消费诊断结果时，可以使用 JSON。JSON 报告采用带版本号的
LSP 风格诊断形状，但仍然只是 CLI 输出契约。

```bash
docs-hygiene check --format json
```

其顶层字段为 `schemaVersion`、`summary` 和 `diagnostics`。每条诊断包含 `source`、
`code`、`severity`、`message`、`path`、`uri`、`range` 和
`relatedInformation`。

规则激活使用独立的版本化 JSON 契约：

```bash
docs-hygiene explain-rules --format json
```

它报告 `docs-hygiene.rule-activation.v1`、项目事实快照和有序规则决策，不执行外部
Adapter。

组合画像契约在相同事实和决策之外报告文档模板绑定覆盖、规范化治理图、Fan 与循环
指标、各维度成熟度与不变量证据：

```bash
docs-hygiene profile --format json
```

其 Schema 为 `docs-hygiene.profile.v1`，顶层字段为 `schemaVersion`、`facts`、
`decisions`、`documentTemplates`、`governanceGraph`、`dimensions`、
`overallObserved` 和 `meetsTargets`。

模板生命周期可以用只读方式检查：

```bash
docs-hygiene migrate-templates --check --format json
```

迁移 Schema 为 `docs-hygiene.template-migration.v1`。不使用 `--check` 时，兼容及缺失的
revision pin 会被原子更新；任何阻断绑定都会阻止策略文件写入。

Kind Schema 与 Template pin 可以作为一个原子计划检查和迁移：

```bash
docs-hygiene migrate-kinds --check --format json
```

报告使用 `docs-hygiene.kind-migration.v1`；任一文档或 Template 绑定不兼容都会同时阻止
文档与策略写入。

## 测试套件

Rust 测试套件直接覆盖 policy engine：

```bash
cargo test
```

当前测试覆盖必需文件、错误命名、重复编号、断号、长度预算、语言阈值、文档契约、
可复用模板合并、注册表故障、兼容迁移与原子阻断、成熟度建议、概念外键、治理 Manifest、水平引用、垂直派生、Library 投影、adapter
失败、豁免匹配、画像迁移、N/A 排除、不变量证据和目标门禁。
同时覆盖规范化边兼容性、锁定边出处、标题 selector 解析、本地化签名同位、Fan/Cycle
指标和阈值诊断，以及带版本的引用 occurrence 收集、显式非语义策略和语法无关规范化。
多锚 fixture 还覆盖 file/block 变化隔离、逐锚来源行、兼容 file JSON 和 opt-in commit
校验；传递影响 fixture 覆盖反向闭包、循环终止和仓库真实影响集合。
身份 fixture 覆盖状态义务、权威后继校验、本地化同位和终止目标陈旧消费诊断。
核心 claim fixture 覆盖权威唯一性、重复策略、受控摘录 block pin 与建议性相似证据。
关键依赖 fixture 覆盖 Pin 策略诊断、反向影响、只读/局部更新、原子阻断和审计记录。

CLI 集成测试覆盖 `--help`、`--version`、`scaffold`、语言策略 CRUD 命令和阻断性
治理关系图故障，也覆盖 claim 候选扫描和审计 Pin 更新。`tests/dogfood.rs` 加载本仓库的真实 `docs-hygiene.yml`，并要求原生
策略和治理关系图通过。

## GitHub Actions

`.github/workflows/ci.yml` 运行完整测试集与 Clippy。测试集包含仓库 dogfood 门禁，
因此断裂的水平引用或垂直派生会阻断工作流。

```yaml
- name: Test, including repository dogfood policy
  run: cargo +1.85.0 test --locked
```

dogfood 测试会关闭外部 adapter，因为它们的可执行文件属于环境依赖；本地
`cargo run -- check --fail-on-warning` 仍会同时检查原生策略和已配置 adapter。

## 后续 CI 能力
