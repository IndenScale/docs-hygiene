# 文档卫生治理模型

本文定义 [PRD-004](intent/prd/prd-004/index.md) 已基线化的模型，用于组织当前已交付
能力，而不是用一个线性成熟度代表所有治理问题。当前可执行行为以配置、规则、测试和
[渐进式规则激活](10_progressive_rule_activation.md)为准。

## 模型

文档卫生包含三个相互独立的轴：

1. 能力维度说明评估哪个治理范围；
2. 成熟度说明该范围治理到多深；
3. 执行状态说明一条适用规则当前如何执行。

分离三者可以避免把仅观察执行、依赖精度和关系图规模误解成一条六层阶梯中的必经
阶段。

## 成熟度

| 等级 | 名称 | 保证 |
| --- | --- | --- |
| G1 | 基础 | 发现输入并确定性地校验基本完整性。 |
| G2 | 受控 | 使策略显式并让变更检测可重复。 |
| G3 | 治理 | 传播影响，并审计策略、预算和例外。 |

三个等级在单个适用维度内累积。仅观察属于执行策略，不是 G0。

## 核心能力维度

| 维度 | G1 基础 | G2 受控 | G3 治理 |
| --- | --- | --- | --- |
| 结构与契约 | 发现、命名、链接 | 文档契约与可复用模板 | 模板版本与结构迁移 |
| 身份与 SSOT | 身份存在和重复检测 | canonical source、语义引用、表示同位 | 生命周期与权威迁移 |
| 依赖与影响 | 引用可以解析 | 类型化边、关键锁定、目标过期检测 | section/block selector、局部哈希、传递影响 |
| 拓扑与规模 | 节点和边指标 | Fan-In、Fan-Out、循环、阈值 | 图预算、公共概念豁免、健康度趋势 |

维度可以不适用。脚手架、编辑器集成和外部 Adapter 是帮助达到成熟度的工具，不是
成熟度维度。

## 统一治理边

语义关系规范化为统一的类型化边模型：

```text
治理边
  ├─ 引用
  ├─ 锁定引用
  ├─ 派生
  └─ 投影
```

这些依赖边类型复用目标解析、selector、锁定、生命周期、变更影响和拓扑策略。标准
Markdown Link 仍是导航性质的路径完整性输入，不能仅因自身是链接就成为语义依赖。

Wiki Link、Markdown Link 和 frontmatter 声明现已共享带版本的 reference occurrence
IR。规范化前，由显式语法与上下文策略判定 occurrence 属于语义依赖、导航输入还是
身份声明；声明语法不再在解析器内部决定边语义。

Wiki Link、`formalizes`、`realizes` 和 `projects` 输入现已规范化到该模型，且没有改变
公开语法和诊断契约。画像会序列化有序边记录、反向传递影响集合及关系图指标；标题
selector、影响与 Fan/循环策略已经消费同一模型，预算和趋势仍属于后续层次。

ATX block、file 与 opt-in repo 锚现已共享同一边模型。块级局部 hash、多锚声明和
确定性传递影响已经交付，因此本仓库依赖维度现已达到治理级。

生命周期状态义务和显式 `supersededBy` 映射现已覆盖资产、Package 领域及叶子。终止
身份不能继续作为当前边目标，声明的替代关系进入画像图；历史转移时间仍由 Git 留证，
不会从当前状态推断。

## 画像与总体等级

文档卫生画像按维度保留目标和检测成熟度。配置接受以下 Schema：

```yaml
hygieneProfile:
  dimensions:
    structure: { target: governed, required: true }
    identity: { target: controlled, required: true }
    dependency: { target: controlled, required: true }
    topology:
      applicability: notApplicable
      rationale: 当前不治理语义依赖图。
```

需要总体等级时，取所有适用且必需维度中的最低结果。总体等级不能替代各维度结果。

## 执行状态

规则执行继续使用 `inactive`、`advisory`、`warning` 和 `error`。项目事实判断适用性并
可以推荐成熟度目标；显式项目策略仍是阻断行为的最终权威。纯规模启发式不会静默地把
非阻断基线提升为 error。

## 交付边界

SPEC-003 已定义画像评估和迁移契约。原子不变量注册、版本化画像输出、N/A 排除、旧值
映射、suppression 不构成证据、统一边、标题 selector 解析、语法中立引用收集、
多粒度多锚校验、传递影响、Fan-In/Fan-Out 与循环分析、显式拓扑阈值与审计例外，以及
责任、复核日落和知识冗余已经交付。当前四级文档
契约成熟度与九个规则族激活模型仍是可执行的兼容基线。
