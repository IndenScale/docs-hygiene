# 源文件体量治理

当不相关职责持续堆积在同一个文件中，代码会变得难以评审，也更难安全修改。因此，
本仓库对 Git 跟踪且未被忽略的文本源文件执行行数预算门禁。

## 阈值

- 超过 500 行产生 warning，要求维护者规划拆分。
- 超过 1,000 行产生 error，并阻断 CI。

warning 只作提示；error 不设置历史豁免。超大文件必须按职责拆分，不能通过永久白名单
绕过治理。

## 本地检查

运行与 CI 相同的门禁：

```bash
./scripts/check-file-size.sh
```

检查器覆盖常见代码、脚本、配置和 Markdown 扩展名。它读取 Git 已跟踪和未忽略的文件
集合，因此不会让构建产物和已忽略依赖产生噪声。

## 模块边界

拆分依据是策略职责，而不是任意行号。核心检查器分别维护包校验、Wiki Link 校验、
派生关系规则、文档契约、仓库检查和共享支持单元；父模块只负责流程编排，并以私有实现
单元的方式组合它们，不扩大 crate 的公共 API。

单元超过 500 行时，应优先提取内聚的数据解析、校验或报告行为，并保留聚焦测试。
超过 1,000 行说明设计需要调整，而不是说明阈值需要提高。
